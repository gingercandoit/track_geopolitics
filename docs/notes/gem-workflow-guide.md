# GEM 地缘政治经济月报：AI 辅助采编工作流指南

> 组会分享用。聚焦"怎么搜、怎么写、怎么避坑"三个核心问题。
> 项目地址：https://geopolitics-tracking.pages.dev/

---

## 一、项目做了什么

用 AI 辅助完成地缘政治经济信息的**采集→筛选→撰写→发布**全链条：

- **5 个议题板块**：制裁与经济管制（T1）、贸易与产业政策（T2）、能源安全与资源角力（T3）、技术竞争与规则制定（T4）、地缘政治信息池（T5）
- **数据驱动**：所有事件存为结构化 JSON，网站从 JSON 自动生成
- **静态站点**：Python+Jinja2 生成纯 HTML，Cloudflare Pages 托管，零服务端

## 二、信息来源分层（核心概念）

| 层级 | 定义 | 示例 | 角色 |
|------|------|------|------|
| **A 层** | 官方政府/监管机构 | 白宫、USTR、BIS、OFAC、中国商务部、外交部、新华社 | 最高权威，**必须覆盖** |
| **B 层** | 国际机构 | IMF、WTO、IEA、OECD、World Bank | 权威数据与框架 |
| **C 层** | 智库与媒体 | FT、Reuters、Bloomberg、PIIE、CFR、CSIS | 分析视角 |
| **D 层** | 社区策展 | Reddit r/geopolitics、X/Twitter | 事件发现信号，**不作为来源引用** |

**关键原则**：
- D 层发现线索 → 回溯到 A/B/C 层获取权威信息
- A 层占比 ≥ 30%（每份月报的硬约束）
- 涉及中方行动的事件，必须有中方 A 层来源直接引用（新华社/商务部/外交部），不可仅依赖西方媒体转述

## 三、怎么搜

### 工具矩阵

| 工具 | 覆盖源 | 适用场景 | 注意事项 |
|------|--------|----------|----------|
| Exa 搜索 | 全网 A/B/C 层 | **主力**：回溯、历史发现、无 RSS 的来源 | 每议题 5-8 组关键词 |
| RSS（Bloomberg/WSJ/政府） | C 层实时 + A 层美国政府 | 补充：最近 24-72 小时事件 | 仅缓存 1-3 天，不能替代 Exa |
| Reddit JSON API | D 层事件发现 | 发现重大事件信号（高票帖） | ≤10 请求/分钟，需自定义 UA |
| requests + trafilatura | 指定 URL 深度抓取 | 读原文、提取正文 | 首选方案（终端执行，无弹窗） |

### 搜索节奏

```
1. RSS/Reddit 拉取（事件发现层）
   ↓
2. Exa 搜索 5-8 组关键词（主力发现）
   ↓
3. 合并去重（URL 为 key）→ 按来源层级排序
   ↓
4. 深度抓取 3-5 篇 A 层来源原文
   ↓
5. 筛选 → 撰写
```

### 关键词设计原则

- 每个议题准备 5-8 组关键词，英文为主
- 同一事件换角度搜：`OFAC sanctions` / `Russia sanctions evasion` / `secondary sanctions compliance`
- **中方视角单独搜**：`site:english.news.cn + sanctions`，不可遗漏
- 结果 < 3 条高质量 → 变换关键词再搜一轮

### 已知盲区

| 来源 | 问题 | 解决 |
|------|------|------|
| IMF/World Bank/IEA/OECD | 无 RSS，RSSHub 全部 403 | 必须用 Exa 搜索 |
| PIIE/Brookings/CFR/CSIS 智库 | RSS 被防火墙封锁 | Exa 或 trafilatura |
| 知乎/微信公众号 | 强制登录 + 反爬 | 放弃自动抓取，手动贴内容 |
| 中国官方 | 仅新华社英文版有 RSS | 商务部/外交部用 Exa 搜索 |

## 四、怎么筛

### 纳入标准

- A/B 层的任何官方声明或政策文件
- C 层中有明确政策含义的深度分析
- 包含可量化指标（关税率、制裁名单数量）
- 涉及多国博弈的多边事件

### 排除标准——30 天法则

> 问自己："这件事在 30 天后还重要吗？它改变了什么结构性的东西？"

排除项：
- 外交威慑/口头警告（"北京警告将采取反制"）
- 安理会投票姿态（弃权票、谴责声明）
- 例行代表团访问
- 当日战况/油价波动
- 推测性分析（"可能导致..."）

**例外**：首次标志结构性转变的事件仍收录（首次制裁、首次封锁）。

## 五、怎么写

### 事件卡片结构

```
导语（20-40 汉字，一句话概括）
• bullet 1：核心事实 + 数字
• bullet 2：背景/影响
• bullet 3：时机/前瞻
来源：[来源名](URL)
```

### 五条铁律

1. **导语 20-40 字**——超出扫不完，不足缺信息
2. **每卡 ≥ 3 条 bullet**——事实 + 背景 + 影响，三维度最低限
3. **导语仅引 A/B 层行动**——咨询报告、企业动态放 bullet，不放导语
4. **谁对谁做了什么**——"美国USTR对76国发起301调查" ✓ / "贸易政策调整" ✗
5. **国别 = 第一信息**——"美国BIS"、"美国OFAC"，机构缩写前必须加国别

### 月度总览格式

```
**一句话断言（加粗，仅 A/B 层行动）。**

- **主体 日期**：具体动作+数字
- **主体 日期**：具体动作+数字
- ...（4-8 条，按日期升序）
```

### 禁用词速查

| 禁止 | 替代 |
|------|------|
| "结构性演变/侵蚀" | 直说发生了什么 |
| "三条主线" | 不框架化，直叙 |
| "标志着…时代的到来" | 删除 |
| "华盛顿""北京" | "美国""中国" |
| 破折号升华（"——中国赢了"） | 删除 |

### 写作风格参考

**财新**——证据链记者：每句话有信源，日期+机构+动作，不判断。
**经济观察报**——利益追踪者：追问"谁受益谁受损"，分析锚在利益关系上。

## 七、最痛的教训

| # | 教训 | 代价 |
|---|------|------|
| 1 | **质量标准首次写入就要达标**——导语字数、bullet 条数在第一次生成就执行，不留返工 | 78/101 事件返工，约 6 小时 |
| 2 | **审计先行**——先跑审计脚本统计不达标数，再制定修复计划 | 无审计 = 盲目修复 |
| 3 | **导语不放分析**——导语是断言（A/B 层行动），不是综合概括（C 层咨询报告） | 用户指出"不够有力" |
| 4 | **中方来源不可替代**——涉中事件必须引用中方 A 层来源，新华社英文版是主力 | 视角偏差 |
| 5 | **事件类型必须用固定词表**——自造类型（"技术管制""市场动态"）导致 10/101 违规 | 全量审计修复 |
| 6 | **缓存破坏必须统一**——所有静态资源（CSS+JS）都要 `?v=timestamp` | 用户看到旧版本 |
