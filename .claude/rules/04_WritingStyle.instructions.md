# 写作风格规范

> 跨所有 Prompt 适用。每次生成文字内容前，必须过一遍这份清单。

---

## 一、禁用词与套话（不得出现）

以下词汇和句式是机器文风的标志，**一律禁止**：

| 禁止                                                   | 原因                                                       |
| ------------------------------------------------------ | ---------------------------------------------------------- |
| "结构性演变/侵蚀/调整/转变"                            | 万能词，等于没说                                           |
| "A+B+C 三条主线" / "可提炼为N条主线"                   | 框架化而非叙述                                             |
| "呈现出…主旋律"                                       | 假装总结，实为空话                                         |
| "与此同时" × 3+                                       | 可以用一次，不可滥用                                       |
| "标志着…时代的到来"                                   | 新闻通稿常见套话                                           |
| "值得关注的是"                                         | 废话引导语                                                 |
| "不可逆转地"                                           | 过度断言                                                   |
| "深层矛盾…在极端场景下被迫暴露"                       | 过度修饰                                                   |
| "折射出…深层不信任"                                   | 类比替代分析                                               |
| "这一趋势正在…"                                       | 模糊时态                                                   |
| "进一步" × 3+                                         | 蹭进展词                                                   |
| `——` + 升华句（破折号强行升华）                    | 如"…大局已定——中国赢了"，这是最典型的机器升华，一律禁止 |
| 滥用符号（`·`、`——`、`…`、`→`、`✓`等） | 符号不是语气，有啥说啥，不要用符号代替文字表达             |
| "华盛顿""白宫"代指美国                                | 英文新闻体修辞，中文直说"美国"                            |
| "中国芯片企业""该机构"等含糊指代                      | 有确切名称时直接使用（"中芯国际""OFAC"）                  |

## 二、写作准则

### 事实优先

- **具体事实先于框架**：先说"财政部临时豁免1.4亿桶伊朗原油"，再说意义；不要先说"制裁工具正面临新挑战"再举例
- **数字胜过形容词**：写"76个经济体"而非"史无前例的广泛范围"
- **动词驱动**：用"USTR发起调查""北京限制出境""BIS撤回草案"，不用"政策工具的调整"
- **谁对谁做了什么**：标题和 bullet 必须交代清楚主体、对象和动作。"欧盟新增制裁"✘ → "欧盟新增31人和6实体到涉俄/涉伊制裁名单"✓
- **确切名称优先**：有具体机构名/企业名/人名时直接使用，不用含糊代称（"中芯国际"✓，"中国芯片企业"✘）
- **国家做主语**：中文报告中用"美国""中国"做主语，不用英文修辞代称（"华盛顿""北京"偶尔可用但不作默认）

### 中文风格参考：财新+经济观察报

**只学这两家，不学其他。** 不学南方周末的场景派叙事（不做文学铺垫、不靠长短句节奏），不学 Economist 的英式俏皮。

**财新**——证据链记者：

- 每句话有信源：日期+机构+动作，不用"近日""有关方面"
- 匿名信源也标层级："一位接近交易方的人士""知情人士"
- 不做判断：没有"这意味着""这标志着"，事实摆出来，读者自己想
- 括号只在首次出现缩写时用一次，后文直接用缩写

**经济观察报**——利益追踪者：

- 始终追问"谁受益、谁受损"，段落围绕利益冲突展开
- 开场从具体的利益矛盾切入，不从框架切入
- 比财新多一层分析，但分析锚在利益关系上，不锚在抽象框架上
- 不用对仗句制造节奏感，直接说事

### 场景映射

| 场景        | 风格              | 要点                                               |
| ----------- | ----------------- | -------------------------------------------------- |
| 事件 bullet | 财新              | 主语+动词+数字，每条有信源归因                     |
| 月度总览    | 经观为主+财新密度 | 每段首句指出利益冲突或政策转折点，后面用证据链支撑 |
| digest 导语 | 经观              | 一句话指出"谁的处境变了"或"哪个均衡被打破了"       |
| 背景参考    | 财新              | 纯事实回溯，不铺垫不渲染                           |

### 英文风格参考（辅助）

- **Reuters**：倒金字塔，最重要的事放第一句；不为读者"总结意义"
- **FT**：机构立场的归因极精确，每个判断都有出处

### 字数纪律

- **主页摘要卡片（digest_zh）**：80-120汉字，不要超
- **月度总览（overview）**：导语1句 + bullet 4-8条，每条≤40字
- **事件bullet**：每条一个独立信息点，不叠加

### 段落结构

- 段落首句 = 核心断言（可以直接引用数字/事件）
- 几个支撑细节
- 可以有一句意义，但要具体（"这是4年来首次突破100美元"）而非空泛（"影响深远"）

### 缩写与机构名

**三种场景，三种处理方式：**

#### 报告正文（Markdown 事件条目）
- **首次出现**：缩写在前，括号内先中文全称再英文全拼，如"CFR（美国外交关系委员会，Council on Foreign Relations）"
- **后文**：直接用缩写

#### 月度总览 bullet
- bullet 中**直接用缩写**，如 `**CFR 3/6**`（总览追求密度和可扫描性）
- 在总览最后用 blockquote 小字注释所有首次出现的缩写：
  ```
  > CFR = Council on Foreign Relations（美国外交关系委员会）
  ```
- 模板会渲染为灰色小字底部注释

#### 事件卡片（JSON → 网站渲染）
- **summary_zh 保持简洁**：直接用缩写，不在行文中展开双语全称
- **卡片底部脚注**：在 JSON 事件对象中添加 `footnotes` 数组：
  ```json
  "footnotes": [
    "CFR = Council on Foreign Relations（美国外交关系委员会）",
    "CIPS = Cross-Border Interbank Payment System（人民币跨境支付系统）"
  ]
  ```
- 模板在来源下方自动渲染为编号灰色小字（1. 2. 3.）
- 只有该缩写**首次出现**的事件需要加 footnotes，后续事件不重复

#### 通用规则
- **读者视角**：假设读者不是本领域专家，不认识 CFR/CIPS/SWIFT 等缩写
- **人名**：首次出现全名+职位，后续可用姓氏

## 三、月度总览格式规范（overview）

> 适用于 `reports/topicN-*/YYYY-MM.md` 中的 `## 月度总览` 部分。

### 结构

```markdown
## 月度总览

**导语——一句话概括本月主线。**

- **来源主体 M/D**：具体动作+数字
- **来源主体 M/D**：具体动作+数字
- ...（4-8条）
```

### 导语

- 加粗，一句话，用编辑断言概括本月最核心的变化
- 不用"呈现N条主线"式框架，直说发生了什么

### Bullet 格式

- 每条格式：`- **来源主体 M/D**：描述`
- **来源主体**：机构名、国家名或具体行为主体（OFAC、欧盟、美财政部、CFR 等）
- **来源主体必须是国家/政府机构/国际组织/企业**：律所、咨询公司、媒体等非行为主体不单独出条，其信息应并入对应政策主体的 bullet（如 Mayer Brown 对 Project Vault 的分析并入"美国EXIM"或"美国白宫"）
- **日期**：必须是**具体单日**（`3/12`），禁止使用日期区间（`3/6–3/19` ✘）
- 如果同一主体多日有动作，拆成多条，每条一个日期
- **中文冒号**：`：`（全角），不用 `:`
- 描述部分：财新风格——主语+动词+数字，不加意义推导

### 排列规则

- **按日期升序**（从月初到月末）
- 同日事件按来源层级排序（A层 > B层 > C层）
- **仅收录当月事件**：非本月发生的事件不放入总览 bullet（可在"背景参考"中交代前置事件）

### 周分组规则（网站展示）

- 月度报告按周展示事件时，周的日期范围按**日历7天分**：
  - 第1周 = 1日–7日
  - 第2周 = 8日–14日
  - 第3周 = 15日–21日
  - 第4周 = 22日–28日
  - 第5周 = 29日–月末（如有）
- 不按事件实际日期的 min-max 计算周范围

## 四、典型改写示例

**原文（禁止）**：

> 2026年3月，制裁与经济管制领域呈现三条交织主线：美国OFAC制裁机器的高频运转、欧盟对俄多层制裁的持续推进，以及人民币/CIPS体系对美元制裁基础设施的结构性侵蚀。

**也禁止（破折号升华、白描叙述风格）**：

> 3月，北京将稀土纳入出口管制清单——中美之间第一次两手都有了武器。

**改写（目标）**：

> 3月，北京将稀土、镓、锗纳入出口管制清单。

## 五、语言禁忌 × digest_zh 专项

主页摘要卡片的额外要求：

**格式规范**：

```
加粗第一句（编辑断言，精简）
>> 具体事实1（纯动词+数字，不加意义解读）
>> 具体事实2
>> 具体事实3
```

在 digest_zh 字段中：

- 第一行 = 加粗的编辑导语，无 `**` 标记（模板自动加粗）
- 后续行以 `>>` 开头 = bullet 事实列表，模板渲染为 `<li>`

**内容原则**：

- **第一句是断言**，用来勾住读者，但不加破折号升华，不加"这标志着"
- **bullet 直说做了啥**：谁做了什么，数字是多少，不加意义推导
- **bullet 要筛选**：只保留有实质政策意义的事件，排除例行通告（如"OFAC发布第N次制裁通知"无具体对象时不列）
- 不照搬月度总览第一句
- 不滥用符号
- **bullet 内绝不使用括号**（中文括号 `（）` 或英文括号 `()`）：括号里的补充信息，要么删除（不重要），要么改写融入句子本身

## 六、文献字段写作规范（data_zh / method_zh）

> 适用于 `literature/new/new.json` 和 `literature/classic/classic.json` 中每篇论文的 `data_zh`（数据来源）和 `method_zh`（方法论）字段。

### abstract_zh：中文摘要精炼翻译

**格式**：2-4句自然语言段落

**原则**：
- **忠于原文摘要**的客观翻译，不是主观简评或推荐语
- 覆盖：研究问题 → 方法/数据 → 核心发现/结论
- 不加"本文""该研究"等冗余引导词，直接切入内容
- 无英文 abstract 的论文：基于标题、已知内容和领域知识撰写
- 与 `notes_zh`（主观简评）区分：`abstract_zh` 回答"这篇论文研究了什么"，`notes_zh` 回答"为什么收录这篇"

### data_zh：纯数据来源列举

**格式**：`XX机构的A数据；YY的B数据`

**原则**：
- 只写**用了什么数据**，不写怎么用、为什么用
- 用分号 `；` 分隔不同数据来源
- 数据名称具体到数据集名（COFER、QCEW、CES），不用泛称（"面板数据""微观数据"）
- 时间范围标注在括号内：`（1999-2022）`
- 无实证数据的论文写`无（理论论文）`或`无（综述论文）`

**示例**：
- `各国央行国际储备币种构成（COFER，1999-2022）；联合国投票分歧、军事联盟和制裁数据；EPU、VIX等不确定性指标`
- `企业财报电话会议记录语料库，多个开源LLM交叉验证`
- `欧洲央行消费者预期调查（CES）微观数据`

### method_zh：类型标签 + 方法描述

**格式**：`{类型标签}。具体方法描述`

**类型标签**（首词，五选一）：
| 标签 | 含义 |
|------|------|
| 模型 | 理论模型、DSGE、一般均衡、博弈等 |
| 实证 | 回归、DID、IV、事件研究、面板等 |
| 实验 | RCT、实验室实验、调查实验等 |
| 综述 | 文献综述、经济思想史、叙述分析 |
| 理论 | 纯理论推导、机制讨论、框架构建 |

**原则**：
- 首词为类型标签，句号后写具体方法
- 用逗号和句号组织句子，不用破折号`——`，不用加号`+`连接
- 常识性英文括号删除（"晶圆厂"不加"(fabrication)"）
- 技术缩写首次出现用括号标注：`双重差分（DID）`
- `vs` 可用，`(1)(2)` 编号可用
- 斜杠 `/` 只在并列缩写中保留（如 JIE/REStat），中文并列用顿号

**示例**：
- `实证。县级双重差分（DID），有半导体设施县vs其他高科技县比较`
- `模型。开放经济新凯恩斯DSGE，纳入中间投入品贸易，区分临时与永久关税的短期宏观效应`
- `实验。RCT，同一提案随机替换合作者国籍为中国vs德国，测量支持率差异`
- `综述。系统整合2018-2025年美国关税实证研究，从新古典贸易理论推导福利和分配效应`
